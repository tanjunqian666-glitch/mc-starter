## 做了什么

### P2.10 崩溃报告上传（API 上报）
- `model/crash_report.go` — CrashReport + UploadRequest/UploadResponse 类型
- `config/config.go` — PostCrashReport 方法（POST 到服务端）
- `server/handlers.go` — 新端点 `POST /api/v1/packs/{name}/crash-report`，存储到文件系统
- `server/server.go` — 注册路由
- `repair/upload.go` — CollectAndUpload 收集崩溃信息（日志尾、崩溃报告、hs_err）+ 上传
- daemon 检测到崩溃时自动静默上传（goroutine，不阻断流程）

- 不适用于 rollback/list/loader-only 操作
- headless 模式也支持自动同步

### P2.13 托盘菜单入口
- `internal/tray/tray.go` — Manager 接口
- `internal/tray/tray_windows.go` — walk NotifyIcon 实现
  - 右键菜单：立即更新 / 修复（完全/模组/配置/回滚） / 打开.minecraft / 退出
  - 崩溃气泡通知
  - 状态同步到悬停提示
- `internal/tray/tray_stub.go` — 非 Windows 空实现
