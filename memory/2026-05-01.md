## P5 启动器感知 — 复盘 + 策略重写

### 背景
原"P5 启动器兼容"目标废弃，改为 **启动器感知 (Launcher Awareness)**。

核心思想：**不兼容启动器，而是感知其存在，利用其配置信息拿到用户惯用的 .minecraft 目录**。
不兼容官方启动器（不开源，无法做），HMCL 暂不投入。

### 两个核心目标
1. **找到启动器** — 自动检测 PCL2 在用户电脑上的位置
2. **从启动器配置里拿到 .minecraft 目录** — 读取 PCL.ini 的 LaunchFolders

### 实际用途
用户可能通过 PCL2 预先下过整合包，里面有自己的设置/截图/存档。我们要接管已有的整合包做增量更新，不是重新下载。

### 设计新增 — sentinel 文件 + 多目录管理

**sentinel 文件标记**：`.minecraft/starter_repo/repo.json` 作为"这是我们的目录"标记。
`IsManaged(mcDir)` 检查标记是否存在。

**多目录场景**：
- 检测到多个 .minecraft 目录时，GUI 变成下拉框让用户选
- 副版本也有独立目录选择框（启用时才显示）
- `LocalConfig.MinecraftDir`(单值) → `MinecraftDirs`(map[string]string, key=包名, val=路径)
- 旧字段 `MinecraftDir` 读时自动迁移到 `MinecraftDirs["_default"]`

**同包名副本冲突解决**（同一包名出现在不同目录）：
- `resolveDir()` 优先级：已记录 > 有标记 > 最新 > 第一个

**同目录相似包名检测**（同一目录下 `main-pack` 和 `main-pack-copy`）：
- `FindSuspectedDuplicates()` 前缀匹配
- **只提示不操作**，宁可漏过不可误吞

### 经验总结（补入）

**GUI ComboBox 在 walk 中的使用：**
- `ComboBox.SetModel([]string)` 设置下拉选项
- `ComboBox.SetCurrentIndex(int)` 选中指定项
- `OnCurrentIndexChanged` 回调监听选中变化
- walk 的 ComboBox 不支持带不同值的 model（不像 Qt 的 QComboBox+userData），所以需维护并行数组 `[]mcDirItem` 作为数据源，用索引同步

**Go 类型系统陷阱（函数内类型 vs 包级类型）：**
- 不能在包 A 的函数内定义 `type X struct` 同名于包级的 `type X struct`
- Go 认为它们是不同 identity 的类型，即使字面量完全一致
- 编译报错："cannot use value of type X as type X" — 极其绕人
- 解决方案：函数内直接用包级类型，不要在函数内重新定义

**WinVM 工具稳定性提升空间：**
- stderr 捕获不完整：PowerShell `2>&1` 在远程 SSH 中被当作错误流而非文本流
- 解决：用 `2>&1 | Out-String` 把错误流转为字符串，或写 .ps1 上去执行
- 同步后编译需要先 `--sync-only` 再执行，否则可能用到旧代码
