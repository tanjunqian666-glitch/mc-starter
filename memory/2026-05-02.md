# 2026-05-02 — GUI 重构：G.1 EventBus + G.2 StateMachine + G.3 ViewModel

## 做了什么

### G.1 EventBus (`internal/gui/eventbus.go`)
- 纯 channel 广播，带缓冲防阻塞
- 同步 `Subscribe()` / `Unsubscribe()` — 阻塞直到 Run loop 处理完
- 快捷 emit 方法：`EmitProgress` / `EmitLog` / `EmitError` / `EmitStateChange` / `EmitSyncDone`
- `Close()` 等待 Run 安全退出
- 11 个测试全通过

### G.2 StateMachine (`internal/gui/state.go`)
- 8 个状态：`Idle → Checking → Downloading → Installing → Done` + `Error` / `Cancelled` / `Repairing`
- 完整转换规则表 + `OnChange` 回调
- 6 个测试全通过

### G.3 ViewModel (`internal/gui/viewmodel.go`)
- 状态容器：版本选择、同步生命周期、进度
- 注入 EventBus，状态机变更自动通知
- `CurrentPackStatus()` 返回带颜色/文案的 UI 状态
- `SaveLocalConfig()` 持久化
- `pierceToLocked()` 穿透中间状态（如 Checking→Done）以跳过全路径
- 7 个测试全通过

### internal/model/copy.go
- 新增 `LocalConfig.Copy()` 深拷贝方法

## 遇到的问题
- **EventBus 时序竞态**：Subscribe 原来异步导致事件丢失 → 改为同步阻塞直到 Run loop 确认注册
- **状态机穿透**：`MarkSyncDone` 需穿透 `Checking → Downloading → Installing → Done`，State 转换规则不允许跳转 → 加 `pierceToLocked()` 自动穿透
- **Windows VM 同步**：sync 后需 `time.Sleep(0.5s)` 确保 IO flush，否则 Go 编译旧代码
- **go vet 踩 CGO**：`go vet ./internal/gui/` 需要 walk (CGO)，在 Windows VM 上用 `go vet <file>` 单文件能过但包级 vet 因 CGO 报 Undefined
