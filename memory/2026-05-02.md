### G.14 — 下拉框切换版本时 UI 联动 ✅
- `onPackSelected()` 改用 `ServerPacksFiltered()`（只返回已启用版本），索引对齐
- 新增 `ensureValidSelection()` — 选中版本被禁用时自动切主版本
- 新增 `syncPackCBSelection()` — 刷下拉时保持选中项与 ViewModel 一致

### G.15 — 副版本启用/禁用时下拉显隐 ✅
- `PackNames()` 过滤：只返回 `Enabled==true` 的包名
- `isPackEnabledLocked()` — 主版本默认启用，副版本未配置时不显示
- `settings.go` 全面重构：
  - 所有副版本控件的 MC 目录行**预构建**（不再根据勾选状态有条件创建）
  - 新增 `subPackUI` 结构体 + `buildSubPackUI()` 构建函数
  - 勾选/取消勾选时通过 `SetVisible()` 动态显隐 MC 目录行
  - 保存时从面板控件读取最终值

## 遇到的问题
- **Walk 不能动态 Add/Remove** — 改为预构建全部控件，SetVisible 控制显隐
- **Walk 老版本 API 差异** — MsgBox 返回 int（不是 (int, error)），Label 无 Wrap 字段
- **settings.go 用 a.Lock() 但 App 没 Mutex** — 补上 `sync.Mutex` 嵌入
- **comboBox 模型必须在 Create 后设置** — buildSubPackUI 只存指针，Create 后 Synchronize 设模型
