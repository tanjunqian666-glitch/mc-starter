# 客户端与服务端通信（API 契约）

> 新增文档 — 2026-04-30

---

## 一、更新流程（客户端侧）

```
starter update
│
├─ 1. 读 local.json
│      → server_url, packs
│
├─ 2. GET /api/v1/ping
│      → 验证服务端可达
│
├─ 3. GET /api/v1/packs
│      → 拿到包列表 + primary 标记
│
├─ 4. 对比本地 packs 状态
│      → 找出需更新的包
│      → 主包: 永远同步
│      → 副包: 只同步 enabled=true 的
│      → 首次连接: 提示副包可用
│
├─ 5. for each 需更新的包:
│      GET /api/v1/packs/{name}/update?from={local_version}
│      → 拿到 {mode, added, updated, removed}
│
│   ├─ 如果 mode == "incremental":
│   │   ├─ 删除 removed 中的文件
│   │   ├─ for added + updated:
│   │   │   ├─ 查 CacheStore 命中 → 复制
│   │   │   └─ 未命中 → GET /files/{hash} → 下载 + 写入 CacheStore
│   │   └─ 更新本地版本号
│   │
│   └─ 如果 mode == "full":
│       └─ 下载全量包 → 解压 → 覆盖
│
├─ 6. 更新 repo 快照
│
└─ 7. 完成
```

## 二、多包同步策略

| 包类型 | update 行为 | update --all | 单独更新 |
|--------|------------|-------------|---------|
| 主包 | ✅ 默认同步 | ✅ | `starter update --pack main-pack` |
| 副包（已启用）| ❌ 不同步 | ✅ | `starter update --pack extra` |
| 副包（已禁用）| ❌ 不同步 | ❌ | 需先启用 |

## 三、错误处理

| HTTP 状态码 | 含义 | 客户端行为 |
|-------------|------|-----------|
| 200 | 正常 | 解析响应 |
| 400 | 请求参数错误 | 打印错误并退出 |
| 401/403 | 认证失败 | 提示 token 无效 |
| 404 | 包/版本/文件不存在 | 降级处理或报错 |
| 413 | 请求体太大 | 提示包过大 |
| 500 | 服务端错误 | 重试 3 次后报错 |

## 四、客户端缓存策略

- **包列表** — 每次 update 都拉（小请求）
- **增量清单** — 每次 update 都拉
- **文件** — CacheStore 按 SHA256 去重，全局共享
- **server.json 缓存** — 本地 local.json 的 packs 字段持久化，可作为离线占位

## 五、重试与降级

```
请求失败 → 等待 1s → 重试 → 等待 3s → 重试 → 等待 10s → 重试
                                                            ↓
                                                    3 次均失败
                                                            ↓
                                              ┌─ 离线模式: 使用本地缓存
                                              └─ 报错: "无法连接服务器"
```
