# 服务端架构与部署（mc-starter-server）

> 新增文档 — 2026-04-30

---

## 一、概述

`mc-starter-server` 是一个独立的服务端进程，通过 REST API 管理多个整合包的版本发布和文件分发。

**定位**：不是 MC 服务器插件，是独立的二进制文件，可在 Windows/Linux 上运行。

**与客户端的关系**：

```
管理员:
  mc-starter-server start
    → 监听 :8443 (HTTPS, 可配)
    → 等待 HTTP API 请求

管理操作:
  curl -X POST https://server:8443/api/v1/admin/packs \
    -H "Authorization: Bearer {token}" \
    -d '{"name":"main-pack","display_name":"主服整合包","primary":true}'

客户端操作:
  starter update
    → GET https://server:8443/api/v1/packs → 拿到包列表
    → 对主包: GET /api/v1/packs/main-pack/update?from=v1.0.0
    → 按 hash 下载: GET /api/v1/packs/main-pack/files/{hash}
```

## 二、部署形态

```
/opt/mc-starter-server/
├── mc-starter-server           ← 单二进制
├── server.yml                  ← 配置文件
├── data/                       ← 数据目录
└── packs/                      ← 整合包存储
    └── index.json
```

**Docker**: 提供 Dockerfile + docker-compose.yml

## 三、配置文件（server.yml）

```yaml
server:
  host: "0.0.0.0"
  port: 8443
  tls_enabled: false            # 建议生产启用
  tls_cert: ""
  tls_key: ""

auth:
  enabled: true
  admin_token: "change-me-please"  # 管理 API 的 token
  client_require_token: false     # 客户端是否需要 token

storage:
  data_dir: "./data"
  packs_dir: "./packs"
  file_storage: "local"           # local | s3 (暂只支持 local)
  max_pack_size_mb: 1024          # 单包上传上限

packs:
  default_primary: ""
```

## 四、REST API 协议

### 客户端端点（无需 token）

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/ping` | GET | 健康检查 |
| `/api/v1/packs` | GET | 获取可用整合包列表 |

**响应示例**：
```json
{
  "packs": [
    {
      "name": "main-pack",
      "display_name": "主服整合包",
      "primary": true,
      "latest_version": "v1.2.0",
      "description": "主服务器玩法资源包",
      "size_mb": 50.2
    },
    {
      "name": "extra-content",
      "display_name": "额外内容包",
      "primary": false,
      "latest_version": "v0.3.1",
      "description": "可选附加模组",
      "size_mb": 12.8
    }
  ]
}
```

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/packs/{name}` | GET | 获取指定包详情 |
| `/api/v1/packs/{name}/update?from={version}` | GET | 获取增量变更清单 |
| `/api/v1/packs/{name}/files/{hash}` | GET | 按 hash 下载文件 |

**增量响应**：
```json
{
  "version": "v1.2.0",
  "from_version": "v1.1.0",
  "mode": "incremental",
  "added": [
    {"path": "mods/iris.jar", "hash": "a1b2c3...", "size": 4096000}
  ],
  "updated": [
    {"path": "mods/sodium.jar", "hash": "d4e5f6...", "size": 5120000}
  ],
  "removed": ["mods/optifine.jar"],
  "total_diff_bytes": 9216000
}
```

### 管理端点（需 token）

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/admin/packs` | POST | 创建新包 |
| `/api/v1/admin/packs/{name}` | DELETE | 删除包（含所有版本）|
| `/api/v1/admin/packs/{name}/config` | GET | 获取包配置 |
| `/api/v1/admin/packs/{name}/config` | PUT | 更新包配置 |
| `/api/v1/admin/packs/{name}/import` | POST | 上传 zip 导入（multipart）|
| `/api/v1/admin/packs/{name}/publish` | POST | 发布当前 draft |
| `/api/v1/admin/packs/{name}/versions` | GET | 版本历史 |
| `/api/v1/admin/packs/{name}/versions/{ver}` | DELETE | 删除指定版本 |

**publish 请求体**：
```json
{
  "message": "更新模组版本",
  "primary": true
}
```

## 五、文件存储

文件按 SHA256 前 2 字符分目录存储：

```
packs/main-pack/files/
├── a1/
│   └── a1b2c3d4e5f6...sha256
├── b2/
│   └── b2c3d4e5f6a1...sha256
└── ...
```

## 六、启动命令

```bash
# 启动
mc-starter-server start --config server.yml

# 启动（使用默认配置）
mc-starter-server start

# 检查配置
mc-starter-server check

# 生成默认配置
mc-starter-server init
```

## 七、Docker

```dockerfile
FROM golang:1.23 AS build
WORKDIR /src
COPY . .
RUN go build -o /server ./cmd/mc-starter-server

FROM debian:bookworm-slim
COPY --from=build /server /usr/local/bin/mc-starter-server
EXPOSE 8443
VOLUME ["/var/mc-starter"]
ENTRYPOINT ["mc-starter-server", "start"]
```

## 八、与现有代码的关系

| 现有代码 | 关系 |
|---------|------|
| `internal/pack/pack.go` | 直接被 server 引用，ImportZip/PublishDraft/ComputeDiff 逻辑复用 |
| `internal/launcher/update.go` | 客户端的 UpdateService 改调 HTTP API 而非读本地文件 |
| `internal/config/config.go` | 客户端保留；服务端新增 `internal/server/config.go` |
