# 更新频道（Channel）设计

> 新增文档 — 2026-05-01
> 设计参考：MCUpdater Channel 系统 + Grass-block 时间戳合并

---

## 一、动机

当前多包管理模式中，每个包（Pack）是一个完整资源目录（mods + config + resourcepacks + shaderpacks）。但实际运维场景经常遇到：

- **玩家 A** 只想要核心玩法模组，不想要光影包（文件大、电脑跑不动）
- **玩家 B** 要光影包但不要额外材质包
- **管理员** 发布一个新频道（如"小游戏模组"），不想影响现有的 mods 组合

这些场景的核心需求是：**同一个包内，不同类别的文件可选安装**。

## 二、核心概念

### 2.1 术语

| 术语 | 定义 |
|------|------|
| **频道（Channel）** | 包内的一组文件分类，有独立的版本追踪和启用/禁用开关 |
| **基础频道** | 每个包自带的 `all` 频道，包含包内全部文件，始终启用 |
| **子频道** | 从基础频道拆分的独立分类（如 `mods-core`、`resourcepacks`、`shaderpacks`） |
| **频道状态** | 客户端每个频道的 `enabled` / `disabled` 状态 |
| **频道版本** | 每个频道独立维护当前版本号（时间戳或 semver） |

### 2.2 与现有包体系的关系

```
服务端:
  packages/
    main-pack/              ← 整合包
      channels/
        all/                ← 基础频道（始终同步全部文件）
          meta.json
          files/
            mods/...
            config/...
            resourcepacks/...
        mods-core/          ← 子频道：仅核心模组
          manifest.json     ← 包含文件列表 + 版本
        shaderpacks/        ← 子频道：仅光影包
          manifest.json
        resourcepacks/      ← 子频道：仅材质包
          manifest.json

客户端:
  packs/
    main-pack/
      mods/...              ← 所有已启用的频道文件合并到此
      config/...
```

> **注意**：频道是服务端概念，所有频道文件最终都写入同一个包目录。不同频道的文件不重复。

## 三、文件分类规范

### 3.1 标准频道分类

| 频道名 | 包含目录 | 典型文件量 | 说明 |
|--------|---------|-----------|------|
| `all` | 全部 | 全部 | 基础频道，必选 |
| `mods-core` | `mods/` 中标记为 core | 小 | 整合包必需的模组 |
| `mods-optional` | `mods/` 中标记为 optional | 中 | 可选模组（小地图、JEI 等） |
| `config` | `config/` | 小 | 配置文件（通常必选） |
| `resourcepacks` | `resourcepacks/` | 中 | 材质/资源包 |
| `shaderpacks` | `shaderpacks/` | 大 | 光影包（文件大、性能开销高） |
| `scripts` | `scripts/` | 小 | KubeJS/CraftTweaker 脚本 |

### 3.2 自定义频道

管理员可以在 publish 时创建自定义频道：

```bash
mc-starter-server pack channel create main-pack --name extra-maps --dirs "maps/"
mc-starter-server pack channel create main-pack --name datapacks --dirs "datapacks/"
```

每个频道关联一到多个目录，服务端在 publish 时自动扫描这些目录下的文件，维护独立的 manifest。

## 四、服务端设计

### 4.1 频道数据结构

```go
// Channel 频道定义（存储为 packages/{pack}/channels/{channel}/meta.json）
type Channel struct {
    Name        string   `json:"name"`         // 频道标识（如 "mods-core"）
    DisplayName string   `json:"display_name"` // 显示名称（如 "核心模组"）
    Description string   `json:"description,omitempty"`
    Dirs        []string `json:"dirs"`         // 关联目录（包内相对路径）
    Required    bool     `json:"required"`     // 是否必选（true=客户端不能禁用）
    Priority    int      `json:"priority"`     // 合并优先级（高优先覆盖低优先）
    Version     string   `json:"version"`      // 当前频道版本
    FileCount   int      `json:"file_count"`
    TotalSize   int64    `json:"total_size"`
}
```

### 4.2 频道 manifest（incremental update 用）

```json
{
  "version": "2026-05-01T10:00:00Z",
  "from_version": "2026-04-30T08:00:00Z",
  "mode": "incremental",
  "files": {
    "mods/iris.jar": {
      "hash": "sha256:abc123...",
      "size": 4096000,
      "channel": "mods-core"
    },
    "mods/c2me.jar": {
      "hash": "sha256:def456...",
      "size": 2048000,
      "channel": "mods-core"
    },
    "shaderpacks/complementary.zip": {
      "hash": "sha256:ghi789...",
      "size": 10485760,
      "channel": "shaderpacks"
    }
  }
}
```

### 4.3 存储结构变更

```
packages/
  {pack-name}/
    pack.json                    ← 包描述（不变）
    channels/                    ← 新增：频道目录
      all/
        meta.json                ← 基础频道配置（自动生成）
        manifest.json            ← 当前版本全量文件清单
        versions/                ← 版本历史
          v1.json
          v2.json
      mods-core/
        meta.json                ← 子频道配置
        manifest.json            ← 该频道文件清单
        versions/
      shaderpacks/
        meta.json
        manifest.json
        versions/
    versions/                    ← 已有版本目录
      published/...
      drafts/...
```

#### GET /api/v1/packs/{name}/update?from={version}&channels=mods-core,shaderpacks

增量更新接口支持 `channels` 参数过滤：

```json
{
  "version": "v1.2.0",
  "from_version": "v1.1.0",
  "mode": "incremental",
  "channels": {
    "all": {"version": "v1.2.0", "changed": true},
    "mods-core": {"version": "v1.2.0", "changed": true},
    "shaderpacks": {"version": "v1.0.0", "changed": false}
  },
  "added": [
    {"path": "mods/new-mod.jar", "hash": "sha256:xxx", "size": 1024, "channel": "mods-core"}
  ],
  "updated": [
    {"path": "mods/updated-mod.jar", "hash": "sha256:yyy", "size": 2048, "channel": "mods-core"}
  ],
  "removed": ["mods/old-mod.jar"]
}
```

客户端上报已启用的频道列表，服务端只返回这些频道的增量变更。

#### 管理端 API 扩展

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/admin/packs/{name}/channels` | GET | 获取包的所有频道配置 |
| `/api/v1/admin/packs/{name}/channels` | POST | 创建新频道 |
| `/api/v1/admin/packs/{name}/channels/{channel}` | PUT | 更新频道配置 |
| `/api/v1/admin/packs/{name}/channels/{channel}` | DELETE | 删除频道 |

### 4.5 发布流程变更

```bash
# 发布时自动创建/更新频道
mc-starter-server pack publish --name main-pack

# 或手动管理频道
mc-starter-server pack channel create main-pack \
  --name shaderpacks \
  --display "光影包" \
  --dirs "shaderpacks/" \
  --required=false

# 推送到频道（手动将文件分配到频道）
mc-starter-server pack channel assign main-pack shaderpacks \
  --files "shaderpacks/complementary.zip,shaderpacks/bsl.zip"
```

## 五、客户端设计

### 5.1 配置变更

```go
// local.json 新增 channels 字段
type LocalConfig struct {
    // ... 现有字段
    Packs map[string]PackStateWithChannels `json:"packs,omitempty"`
}

type PackStateWithChannels struct {
    Enabled      bool                `json:"enabled"`
    Status       string              `json:"status"`
    LocalVersion string              `json:"local_version"`
    Dir          string              `json:"dir"`
    Channels     map[string]string   `json:"channels,omitempty"`
    //                      ↑ channel名 → 该频道本地版本号
}
```

示例：
```json
{
  "minecraft_dir": "C:\\Users\\player\\.minecraft",
  "packs": {
    "main-pack": {
      "enabled": true,
      "status": "synced",
      "local_version": "v1.2.0",
      "dir": "packs/main-pack",
      "channels": {
        "all": "v1.2.0",
        "mods-core": "v1.2.0",
        "shaderpacks": ""
      }
    }
  }
}
```

空字符串 `""` 表示该频道未安装。

### 5.2 同步流程变更

```
starter update
│
├─ 1. GET /api/v1/packs
│      → 包列表 + 每个包的 channels 信息
│
├─ 2. for each 已启用的包:
│   ├─ 2a. GET /api/v1/packs/{name}/channels
│   │       → 拿到所有频道的 metadata (required, size, version)
│   │
│   ├─ 2b. 对比本地频道启用状态
│   │   ├─ 新出现的基础频道 → 自动启用
│   │   ├─ 新出现的必选频道 → 无需确认直接启用
│   │   └─ 新出现的可选频道 → 不启用，记录待通知
│   │
│   └─ 2c. GET /api/v1/packs/{name}/update
│           ?from={local_version}
│           &channels=mods-core,shaderpacks  ← 只传已启用的频道
│           → 拿到增量清单 → 下载文件
│
├─ 3. 文件合并到包目录
│      (不同频道的文件写入同一个 packs/{name}/ 目录)
│
└─ 4. 更新本地频道版本号
```

### 5.3 文件冲突处理

不同频道可能包含同名文件（如 `all` 频道和 `shaderpacks` 频道都包含 `shaderpacks/bsl.zip`）。

| 场景 | 处理方式 |
|------|---------|
| 同一个文件出现在多个频道 | 内容一致（SHA256 相同）→ 跳过 |
| 不同频道有同名但不同内容的文件 | 按优先级保留高的，日志警告 |
| 启用/禁用频道时的文件增减 | 禁用频道 → 仅删除该频道的独有文件（不删被其他频道引用的文件） |

### 5.4 启用/禁用逻辑

```
启用频道:
  1. 记录频道版本号到 local.json
  2. 拉取该频道的 manifest，下载缺失文件
  3. 文件写入 packs/{name}/
  4. 完成

禁用频道:
  1. 从 local.json 移除频道版本记录（置空）
  2. 扫描该频道 manifest 中的文件列表
  3. 只删除不被其他已启用频道引用的文件
  4. 完成
```

## 六、服务端发布流程完整示例

### 6.1 管理员准备包目录

```
pack-source/
├── mods/
│   ├── essential.jar        ← 核心模组 → mods-core
│   ├── sodium.jar            ← 核心模组 → mods-core
│   ├── iris.jar              ← 核心模组 → mods-core
│   ├── minimap.jar           ← 可选模组 → mods-optional
│   └── jei.jar               ← 可选模组 → mods-optional
├── config/
│   └── ...
├── resourcepacks/
│   ├── faithful.zip
│   └── xray.zip
└── shaderpacks/
    ├── complementary.zip
    └── bsl.zip
```

### 6.2 导入并分频道

```bash
# 导入包
mc-starter-server pack import main-pack ./pack-source.zip

# 创建频道
mc-starter-server pack channel create main-pack \
  --name mods-core \
  --display "核心模组" \
  --dirs "mods/" \
  --required true

# 分配文件到频道（按文件名匹配）
mc-starter-server pack channel assign main-pack mods-core \
  --files "mods/essential.jar,mods/sodium.jar,mods/iris.jar"

mc-starter-server pack channel create main-pack \
  --name shaderpacks \
  --display "光影包" \
  --dirs "shaderpacks/" \
  --required false

mc-starter-server pack channel assign main-pack shaderpacks \
  --files "shaderpacks/complementary.zip,shaderpacks/bsl.zip"

# 发布
mc-starter-server pack publish --name main-pack
```

## 七、与 P6 WBS 映射

| WBS ID | 任务 | 预估 | 说明 |
|--------|------|------|------|
| P6.1 | 频道数据结构 + 配置格式 | 2h | `Channel` 结构体、`meta.json` 格式、`local.json` 扩展 |
| P6.2 | 服务端多频道管理 | 3h | 频道 CRUD API、publish 时自动/手动分配、manifest 生成 |
| P6.3 | 客户端频道选择 | 2h | `starter channel` CLI、local.json channels 字段读写 |

## 八、开放问题

1. **频道文件分配方式**：自动（按目录规则）vs 手动（管理端指定）→ **P6 先用手动 + 按目录自动分配并存**
2. **频道间文件去重**：是否在服务端存储级别去重？→ **P6 先不做服务端去重，客户端侧处理**
3. **频道版本号格式**：时间戳（MCUpdater 方案）vs semver → **P6 用 semver，与现有体系一致**
4. **增量更新粒度**：按频道还是按整体包？→ **按频道，客户端传已启用频道列表，服务端只返回那部分**
5. **迁移路径**：已有包没有频道结构 → 自动生成 `all` 频道，向后兼容
