# MC 版本更新器 — 自更新方案

> 更新器自身的版本更新管理：检查、下载、替换、回滚

---

## 一、设计原则

```
1. 自更新 = 更新器下载新版本，替换自身 exe
2. 永不损坏：更新失败时能自动回滚到旧版本
3. 无感：后台静默检查，用户使用时才更新
4. 多通道：stable / beta / dev 分级别推送
```

---

## 二、更新检查

### 2.1 更新端点

服务端提供一个 JSON 文件，描述最新版本和下载地址。

```
GET https://你的服务器/api/updater/version.json
```

```json
{
  "version": "1.2.0",
  "channel": "stable",
  "release_date": "2026-04-29",
  "download_url": "https://你的服务器/downloads/starter-windows-amd64-1.2.0.exe",
  "hash": "sha256:abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890",
  "signature": "base64_of_ed25519_signature...",
  "changelog": [
    "新增 Fabric 安装支持",
    "新增本地版本仓库（增量同步）",
    "修复崩溃检测日志轮询内存泄漏",
    "改进 PCL2 卡片注入算法"
  ],
  "min_version": "1.0.0",
  "critical": false
}
```

### 2.2 版本号规范

```
semver 格式：主版本.次版本.修订号
  主版本：不兼容的 API 变更
  次版本：向下兼容的新功能
  修订号：向下兼容的问题修复

  预发布标签（可选）：-alpha、-beta、-rc.1
  例如：1.2.0-beta.1
```

### 2.3 检查时机

```go
type UpdateCheckMode int

const (
    CheckOnLaunch   UpdateCheckMode = iota  // 每次启动时检查
    CheckDaily                              // 每天一次（避免每次启动都检查）
    CheckManual                             // 仅手动检查（`self-update check`）
)
```

默认用 `CheckDaily`：

```go
func shouldCheckUpdate() bool {
    state := loadUpdateState()
    
    // 默认每天检查不超过一次
    if time.Since(state.LastCheckTime) < 24*time.Hour {
        return false
    }
    
    // 强制检查标记
    if state.ForceCheck {
        return true
    }
    
    return true
}
```

## 三、下载更新

### 3.1 下载流程

```go
func downloadUpdate(meta UpdateMeta) error {
    // 1. 创建临时目录
    tmpDir := filepath.Join(".local", "update_tmp")
    os.MkdirAll(tmpDir, 0755)
    
    // 2. 下载新版本
    tmpPath := filepath.Join(tmpDir, fmt.Sprintf("starter-%s.exe.tmp", meta.Version))
    err := downloadFile(meta.DownloadURL, tmpPath, meta.Hash)
    if err != nil {
        return fmt.Errorf("下载失败: %w", err)
    }
    
    // 3. 校验 hash
    if !verifyFileHash(tmpPath, meta.Hash) {
        os.Remove(tmpPath)
        return fmt.Errorf("hash 校验失败，文件可能被篡改")
    }
    
    // 4. 校验签名（如果签名可用）
    if meta.Signature != "" && meta.PublicKey != "" {
        if !verifySignature(tmpPath, meta.Signature, meta.PublicKey) {
            os.Remove(tmpPath)
            return fmt.Errorf("数字签名校验失败")
        }
    }
    
    // 5. 重命名为 .exe（去掉 .tmp）
    exePath := strings.TrimSuffix(tmpPath, ".tmp")
    os.Rename(tmpPath, exePath)
    
    // 6. 保存状态
    savePendingUpdate(meta.Version, exePath)
    
    return nil
}
```

### 3.2 更新禁用机制

如果更新器有更新的版本，但用户当前版本低于 `min_version`，强制更新：

```json
// version.json 中
"min_version": "1.0.0",
"critical": true
```

```go
func checkCriticalUpdate() {
    if meta.Critical && semver.Compare(localVersion, meta.MinVersion) < 0 {
        fmt.Println("⚠  当前版本过于陈旧，需要强制更新")
        fmt.Println("更新内容:")
        for _, line := range meta.Changelog {
            fmt.Printf("  • %s\n", line)
        }
        downloadUpdate(meta)
        applyUpdate()
    }
}
```

## 四、替换与重启

### 4.1 Windows 替换策略

Windows 上不能直接删除正在运行的 exe。有两种方案：

**方案 A：批处理脚本（推荐）**

```go
func applyUpdate() error {
    exePath := getCurrentExePath()
    updatePath := filepath.Join(".local", "update_tmp", fmt.Sprintf(
        "starter-%s.exe", pending.Version))
    backupPath := exePath + ".old"
    
    // 生成更新脚本
    script := fmt.Sprintf(`@echo off
:wait
tasklist /FI "IMAGENAME eq starter.exe" 2>NUL | find /I "starter.exe" >NUL
if not errorlevel 1 (
    timeout /T 1 /NOBREAK >NUL
    goto wait
)
copy /Y "%s" "%s"
if errorlevel 1 (
    echo 替换失败
    pause
    exit /b 1
)
del /F "%s"
start "" "%s"
del "%%~f0"
`, updatePath, exePath, updatePath, exePath)
    
    // 写到临时 bat 文件
    batPath := filepath.Join(os.TempDir(), "starter-update.bat")
    os.WriteFile(batPath, []byte(script), 0755)
    
    // 启动 bat 脚本（继承当前工作目录）
    exec.Command("cmd", "/C", batPath).Start()
    
    // 当前进程退出
    os.Exit(0)
    return nil
}
```

**方案 B：MoveFileEx 延期删除（更优雅但仅 Windows）**

```go
// Windows 原生 API
//
// MoveFileEx 可以在进程退出后：
//   1. 删除旧 exe（DELETE_ON_CLOSE）
//   2. 重命名新 exe 到旧 exe 位置（RENAME）
//
// 不需要 bat 脚本，但需要 cgo
func applyUpdateWindows() error {
    exePath := getCurrentExePath()
    updatePath := filepath.Join(".local", "update_tmp", fmt.Sprintf(
        "starter-%s.exe", pending.Version))
    backupPath := exePath + ".old"
    
    // Step 1: 备份旧 exe
    CopyFile(exePath, backupPath)
    
    // Step 2: 标记新 exe 将在重启时替换旧 exe
    // MoveFileEx(updatePath, exePath, MOVEFILE_REPLACE_EXISTING | MOVEFILE_DELAY_UNTIL_REBOOT)
    // 
    // 等进程退出后，系统会自动完成替换
    // 但需要管理员权限...
    
    // 推荐方案 A，bat 脚本更可靠
}
```

```go
// Windows: 使用 bat 脚本替换+重启
func applyUpdateUnix() error {
    exePath := getCurrentExePath()
    
    // 1. 备份旧文件
    os.Rename(exePath, exePath+".old")
    
    // 2. 移动新文件到位
    os.Rename(updatePath, exePath)
    os.Chmod(exePath, 0755)
    
    // 3. 重启
    exec.Command(exePath, os.Args[1:]...).Start()
    os.Exit(0)
    return nil
}
```

## 五、回滚

### 5.1 自动回滚

```go
func autoRollbackOnFailure() {
    // 新版本首次启动时
    state := loadUpdateState()
    
    if state.PendingUpdate != nil && state.PendingUpdate.Applied {
        // 记录启动成功标记
        state.PendingUpdate.AppliedOk = true
        state.PendingUpdate.AppliedTime = time.Now()
        saveUpdateState(state)
        
        // 如果 10 秒后还没被标记为 success，说明可能崩了
        time.AfterFunc(10*time.Second, func() {
            if !state.PendingUpdate.StartupOK {
                // 启动似乎失败，回滚
                rollbackUpdate()
            }
        })
    }
}

func markStartupOK() {
    state := loadUpdateState()
    state.PendingUpdate.StartupOK = true
    saveUpdateState(state)
}
```

### 5.2 手动回滚

```go
func manualRollback() error {
    exePath := getCurrentExePath()
    backupPath := exePath + ".old"
    
    if _, err := os.Stat(backupPath); os.IsNotExist(err) {
        return fmt.Errorf("没有可用的备份（%s 不存在）", backupPath)
    }
    
    // 先备份当前版本
    currentBackup := exePath + ".rollback-backup"
    copyFile(exePath, currentBackup)
    
    // 回滚
    copyFile(backupPath, exePath)
    
    fmt.Printf("✅ 已回滚到上一个版本\n")
    fmt.Printf("   当前版本已备份到: %s\n", currentBackup)
    
    return nil
}
```

### 5.3 回滚 CLI

```bash
# 手动回滚
starter self-update --rollback
  → 检查 starter.exe.old 是否存在
  → 如果存在：替换并重启
  → 如果不存在：报错

# 查看更新历史
starter self-update --history
  → 1.0.0 → 1.1.0 (2026-04-20) ✓
  → 1.1.0 → 1.2.0 (2026-04-29) ✓  ← 当前

# 指定回滚到某个版本
starter self-update --rollback 1.1.0
  → 使用本地缓存中的 1.1.0 版本（需提前下载保留）
```

## 六、多通道

### 6.1 通道定义

```
stable: 经过完整测试的稳定版本，默认通道，自动更新
beta:   新功能预览版，非稳定版本，可选更新
dev:    开发中版本，手动安装
```

### 6.2 通道配置

```json
// local.json
{
  "update_channel": "stable"    // stable | beta | dev
}
```

### 6.3 各通道 endpoint

```go
func getUpdateEndpoint(channel string) string {
    switch channel {
    case "stable":
        return "https://你的服务器/api/updater/stable/version.json"
    case "beta":
        return "https://你的服务器/api/updater/beta/version.json"
    case "dev":
        return "https://你的服务器/api/updater/dev/version.json"
    default:
        return "https://你的服务器/api/updater/stable/version.json"
    }
}
```

### 6.4 通道升级

```go
// 允许的通道升级路径
// stable ←→ beta（双向切换）
// beta → dev（单向降级到 dev，dev 不能直接升 stable）
// dev → stable 需要先走 beta 过渡

func validateChannelSwitch(from, to string) error {
    if from == to {
        return nil
    }
    
    switch from {
    case "stable":
        if to == "beta" || to == "dev" {
            return nil
        }
    case "beta":
        if to == "stable" || to == "dev" {
            return nil
        }
    case "dev":
        if to == "beta" {
            return nil
        }
        return fmt.Errorf("dev → stable 需要先切换到 beta")
    }
    
    return fmt.Errorf("不允许的通道切换: %s → %s", from, to)
}
```

## 七、交互与通知

### 7.1 启动时检查交互

```go
// 非关键更新：后台下载，下次启动时提示
func checkUpdateSilent() {
    meta, err := fetchUpdateMeta()
    if err != nil || meta == nil {
        return
    }
    
    if semver.Compare(localVersion, meta.Version) >= 0 {
        return // 已经是最新
    }
    
    // 后台静默下载
    err = downloadUpdate(*meta)
    if err != nil {
        log("后台更新下载失败: %v", err)
        return
    }
    
    log("更新 %s 已下载，下次启动时应用", meta.Version)
}
```

### 7.2 更新生效时机

```go
// 启动时检查是否有待应用的更新
func checkPendingUpdate() {
    state := loadUpdateState()
    
    if state.PendingUpdate == nil || !state.PendingUpdate.Downloaded {
        return
    }
    
    // 交互模式：询问用户
    fmt.Printf("📦 更新 %s 已下载\n", state.PendingUpdate.Version)
    for _, line := range state.PendingUpdate.Changelog {
        fmt.Printf("   • %s\n", line)
    }
    
    if confirm("现在应用更新并重启？") {
        applyUpdate()
    } else {
        fmt.Println("下次启动时再应用")
    }
}
```

### 7.3 后台下载 + 下次生效（无感模式）

```go
// 当用户运行 starter run（全自动模式）时
// 启动后检查更新 → 后台下载 → 下次启动时弹出询问
// 这样用户不会因为更新而中断当前流程

func runWithSilentUpdate() {
    // 检查待更新
    checkPendingUpdate()
    
    // 后台检查更新
    go func() {
        meta, err := fetchUpdateMeta()
        if err == nil && meta.Version > localVersion {
            downloadUpdate(*meta)
        }
    }()
    
    // 继续正常流程
    doRun()
}
```

## 八、更新过程中的错误处理

| 错误 | 行为 |
|---|---|
| 下载时网络断开 | 保留已下载部分，下次续传 |
| Hash 校验失败 | 删除临时文件，报错，不退出现有版本 |
| 签名校验失败 | 同上，不信任未签名文件 |
| 替换 exe 失败（权限不足） | 提示以管理员身份运行 |
| 新版本启动后崩了 | 10 秒无正常标记 → 自动回滚 |
| 旧版本备份存在但回滚失败 | 反馈给服务器，提示手动处理 |
| 磁盘空间不足 | 检查可用空间 ≥ 2× 更新包大小，不足则报错 |
