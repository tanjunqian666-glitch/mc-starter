### ❌ 发现并修复

#### 1. gofmt 缩进不一致
- **问题**: 15 个文件不符合 gofmt 规范（主要是 `cmd/starter/main.go:781` 一处缩进用空格代替 tab）
- **修复方法**: 全局 `gofmt -w`
- **教训**: 每次提交前跑 `gofmt -l .`，或者配 pre-commit hook

#### 2. 错误忽略（已确认可接受）
- `os.Remove`, `os.Rename`, `defer Close()` 的错误在某些地方被忽略
- 这是 Go 社区的常见做法：清理/关闭失败非致命，记录日志即可
- 但部分地方有改进空间：

| 位置 | 当前行为 | 建议 |
|------|---------|------|
| `defer os.Remove(tmpFile)` (asset.go:84) | 忽略错误 | OK，临时文件清理 |
| `defer reader.Close()` (library.go:595) | 忽略 | ⚠️ zip 读取器关闭失败应至少 Warn |
| `os.Remove(localPath)` (library.go:453) | 忽略 | OK，SHA1 校验失败后清理 |
| `os.Rename(tmpPath, path)` (cache.go:406) | 忽略 | ⚠️ 原子重命名失败应 Warn |
| `os.Rename(tmpPath, s.filePath)` (sync_state.go:282) | 忽略 | ⚠️ 同上 |
| `os.Remove(s.filePath)` (sync_state.go:238) | 忽略 | OK，状态清理非致命 |

#### 3. main.go 命令处理的安全问题
- `os.Args[1]` 没有先检查 `len(os.Args)`（已在外层检查 `if len(os.Args) < 2`）
- `os.Args[2:]` 直接传索引，对 `run` 等命令安全（flags 有 ExitOnError）
- **状态**: 实际安全，但 `printUsage()` 不包含子命令的完整选项文档

#### 4. "hardcoded" Windows 字符串
- `library.go` 中大量 `"windows"` 字符串判断
- **状态**: 这是 Windows-only 项目的设计选择，不是问题。但如果以后需要跨平台，这些地方要抽象
