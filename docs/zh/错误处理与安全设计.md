# MC 版本更新器 — 错误处理与安全设计

---

## 一、错误处理策略

### 1.1 错误分层

```
用户可见的错误         →  友好中文提示
技术日志错误           →  英文 + 堆栈（debug 级别）
静默错误              →  仅记录到日志文件，不打扰用户
```

### 1.2 错误类型定义

```go
type StarterError struct {
    Code       int           // 退出码
    UserMsg    string        // 给用户看的中文消息
    LogMsg     string        // 日志中记录的技术信息
    CanRetry   bool          // 是否可以重试
    Suggestion string        // 建议操作
}
```

错误码速查：

| 退出码 | 含义 | 典型场景 |
|---|---|---|
| 0 | 成功 | — |
| 1 | 通用错误 | 未知异常 |
| 2 | Java 未找到 | `JAVA_HOME` 和 `PATH` 都找不到 java |
| 3 | 配置错误 | server.json 格式错误 / 字段缺失 |
| 4 | 网络错误 | 所有镜像全部不可达 |
| 5 | 磁盘空间不足 | 剩余空间 < 预估下载大小 |
| 6 | 版本不匹配 | server.json 指定版本在版本清单中找不到 |
| 7 | Hash 校验失败 | 下载文件 SHA1 不匹配 |
| 8 | 权限不足 | 无法写入 .minecraft 目录 |
| 126 | 需要确认 | 危险操作（如覆盖已有 .minecraft） |
| 130 | 用户取消 | Ctrl+C |

### 1.3 网络错误处理

**超时策略**：
```go
var DownloadTimeout = struct {
    Connect    time.Duration = 30 * time.Second  // 连接超时
    Overall    time.Duration = 120 * time.Second // 单个文件整体超时
    BatchTotal time.Duration = 600 * time.Second // 整批下载总超时（Asset 阶段）
}{}
```

**重试策略**：
```
网络错误 → 重试 3 次，指数退避（1s → 3s → 9s）
Hash 校验失败 → 重新下载最大 2 次
HTTP 4xx → 不重试（客户端错误）
HTTP 5xx → 按网络错误处理
```

**所有镜像失败时的行为**：
```
1. 记录所有失败的镜像及其错误原因
2. 输出给用户的错误消息：
   "[ERROR] 无法连接到任何下载服务器
    已尝试: mojang.com (timeout), bmclapi (502)
    请检查网络连接后重试"
3. 使用缓存版本继续（如果有缓存）
```

```go
// 下载前检查
func checkDiskSpace(path string, needed int64) error {
    var stat syscall.Statfs_t
    syscall.Statfs(path, &stat)
    free := stat.Bavail * int64(stat.Bsize)
    if free < needed {
        return &StarterError{
            Code:     5,
            UserMsg:  fmt.Sprintf("磁盘空间不足：需要 %.1f GB，可用 %.1f GB", 
                      float64(needed)/1e9, float64(free)/1e9),
            LogMsg:   fmt.Sprintf("disk full: need=%d free=%d path=%s", needed, free, path),
        }
    }
    return nil
}
```

## 二、安全设计

### 2.1 下载文件完整性校验

所有下载必须校验 hash，不校验就等于把用户当肉鸡：

```go
type HashType string

const (
    SHA1   HashType = "sha1"
    SHA256 HashType = "sha256"
    SHA512 HashType = "sha512"
)

func verifyHash(path string, expectedHash string, hashType HashType) error {
    f, err := os.Open(path)
    if err != nil {
        return err
    }
    defer f.Close()
    
    h := hashType.New()
    if _, err := io.Copy(h, f); err != nil {
        return err
    }
    
    actual := hex.EncodeToString(h.Sum(nil))
    if !strings.EqualFold(actual, expectedHash) {
        return &StarterError{
            Code: 7,
            UserMsg: "文件校验失败，将重新下载",
            LogMsg: fmt.Sprintf("hash mismatch: expected=%s actual=%s path=%s", expectedHash, actual, path),
            CanRetry: true,
        }
    }
    return nil
}
```

**hash 来源优先级**：
1. Mojang version.json 中直接提供的 `downloads.*.sha1`
2. Mojang assetIndex 中每个 object 的 `hash`（SHA1）
3. server.json 中配置的 `modpack.hash`（SHA256）
4. 自更新的 `version.json` 中 `hash`（SHA256）

### 2.2 server.json 防篡改（可选增强）

如果 server.json 通过 HTTP 分发但无 HTTPS，攻击者可注入恶意配置导致下载恶意文件。建议：

**方案 A：HTTPS 分发（推荐，简单有效）**
- server.json 通过 HTTPS 提供
- 所有下载 URL 使用 HTTPS
- 不做额外签名

**方案 B：签名验证（需要额外搭建签名系统）**
```json
{
  "mc_version": "1.20.1",
  "loader": { ... },
  "signature": "base64_encoded_rsa_signature",
  "signer": "mc-starter/v1"
}
```
- 服务端用 RSA 私钥对配置签名
- 客户端启动时用内置公钥验证
- 验证失败则提示用户并拒绝覆盖本地配置

**MVP 阶段建议**：直接用 HTTPS 分发，不搞签名。

### 2.3 自更新安全

```
1. 获取 version.json（HTTPS）
2. 对比版本号
3. 下载新 exe 到临时文件（HTTPS）
4. 校验 SHA256（version.json 中提供）
5. hash 匹配 → 替换
6. hash 不匹配 → 删除临时文件，报错
```

**回滚机制**：
```
替换前：
  starter.exe           ← 当前版本
  starter.exe.new       ← 新版本（已下载校验完成）

替换过程：
  1. 复制 starter.exe  →  starter.exe.bak
  2. 移动 starter.exe.new  →  starter.exe
  3. 启动新版本
  4. 新版本启动失败 → 自动还原 starter.exe.bak

说明：这是一种"两阶段提交"式的更新，即使替换过程中断，
     重启时也能通过检测 starter.exe.new 和 starter.exe.bak 状态恢复。
```

### 2.4 路径穿越防护

```go
// 所有文件路径必须限制在 workDir 范围内
func safeJoin(baseDir, target string) (string, error) {
    cleanBase := filepath.Clean(baseDir)
    cleanTarget := filepath.Clean(target)
    
    // 禁止绝对路径
    if filepath.IsAbs(cleanTarget) {
        return "", fmt.Errorf("absolute path not allowed: %s", target)
    }
    
    // 禁止 ..
    if strings.Contains(cleanTarget, "..") {
        return "", fmt.Errorf("path traversal detected: %s", target)
    }
    
    fullPath := filepath.Join(cleanBase, cleanTarget)
    
    // 最终路径必须在 baseDir 下
    if !strings.HasPrefix(fullPath, cleanBase) {
        return "", fmt.Errorf("path escape detected: %s -> %s", target, fullPath)
    }
    
    return fullPath, nil
}

// 使用示例：下载 libraries 时
remotePath := library.Downloads.Artifact.Path  // 从 Mojang API 获取
localPath, err := safeJoin(mcDir, filepath.Join("libraries", remotePath))
```

### 2.5 用户数据保护

- ❌ 不收集任何使用数据
- ❌ 不发送网络请求以外的任何信息
- ❌ 不读取 .minecraft 以外的用户文件
- ✅ 所有配置和缓存仅在 `workDir` 内
- ✅ 日志默认不记录用户名和路径中的用户名部分（可配置）


## 三、总结：安全原则

1. **所有下载校验 hash** — 没有例外
2. **所有路径限制在 workDir 内** — 路径穿越是常见漏洞
3. **HTTPS 分发配置** — 不自建签名系统增加复杂度
4. **自更新两阶段提交** — 防止替换过程中断导致程序损坏
5. **不收集用户数据** — 信任是用出来的
6. **子进程隔离** — starter 退出时确保 MC 也退出
7. **最低权限运行** — 不需要管理员权限（除非安装到 Program Files）
