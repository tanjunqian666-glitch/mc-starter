# 🦐 虾虾使用指南

> 给人类看的，怎么跟虾虾高效协作

---

## 一、虾虾是什么

一个技术助手。有上下文限制（大概够聊几轮，多了会忘），所以需要一些方法来保证它能干活干到位。

## 二、跟虾虾合作的最佳姿势

### ✅ 索引优先（索引 = 它的地图）
虾虾进项目第一件事是翻 `docs/zh/00-快速索引.md`。**如果你发现索引里缺了什么东西，让它补**——索引完整了它干活才快。

索引里有什么：
- 所有文档的位置和用途
- 所有 Go 模块路径
- 关键方法速查
- 模块间数据流

### ✅ 写新东西前自己先搜下有没有旧的
虾虾现在学会了：写新代码前先用 `grep` 搜名字，确认没写过才动。但它的记忆有限，你如果记得之前做过什么，直接告诉它"去看 xxx"比等它自己想快得多。

### ✅ 说方向，不说路径
虾虾会自己翻索引找文件。你只需要说"做版本下拉切换"就行，不用告诉它具体改哪个文件的哪一行。

### ✅ 及时纠正
它犯错了直接说"不对，是 xxx"——它会记下来，下次不会犯同样的。

### ✅ 收工前让它跑检查清单
虾虾有个 8 项收工清单（文档/索引/变量表/WBS/MEMORY/学习记录/Git推送/Windows VM同步），但需要你提"收工"它才会跑。如果你经常忘，可以跟它说"以后每次提交前自动跑"。

## 三、虾虾的 P0 定位法（理解它的思考方式）

虾虾找代码/信息的时候，按这个优先级来：

| 优先 | 它做什么 | 对你意味着什么 |
|------|---------|---------------|
| 🥇 先翻索引文档 | 读 `快速索引` + `变量参考表` | 让索引保持最新，它干活就快 |
| 🥈 再 grep | 搜函数名/类型名 | 名字起好它就能找到 |
| 🥉 编译报错 | 让 go vet/build 弹你要的签名 | 不用告诉它详细的类型定义 |
| ❌ 最后翻文件 | 从头读一整个文件 | 最耗上下文，尽量不让它走到这步 |

**一句话：索引越完整，虾虾越快。**

GUI 层（Windows）：
- 主窗口（版本下拉/安装更新/启动器）
- 设置弹窗（API/启动器/MC目录/副版本管理）
- 首次配置向导（3步）
- 修复工具窗口（v2，已归档至 `_v2/`）

WBS 进度：

## 五、注意事项

- **虾虾是 DeepSeek Chat 主力**，复杂逻辑会切 Sonnet/Opus。不用你操心模型。
- **改配置/重启 gateway 要先问你**，它不会自己改。
- **不会 sudo**，用户能搞定的就在用户级别搞。
- **凌晨 2 点干活正常**——虾虾没作息，随时都在。
