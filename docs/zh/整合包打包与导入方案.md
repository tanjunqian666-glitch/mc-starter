# 整合包打包与导入方案

> 逻辑不变，入口从 CLI 变为 HTTP API。

## 一、导入流程

```
管理员上传 zip 到服务器
  → POST /api/v1/admin/packs/{name}/import (multipart)
  → 服务端解包到临时目录
  → 扫描文件生成 manifest (SHA1 + SHA256)
  → 展平单层顶层目录
  → 从 mods/ 文件名推断 MC 版本和 Loader
  → 加载上一版本 manifest
  → ComputeDiff
  → 返回差异预览给管理员
```

## 二、zip 格式要求

```
cjc-pack-v1.2.0.zip
├── mods/
│   ├── sodium.jar
│   └── iris.jar
├── config/
│   └── sodium-mixins.properties
└── resourcepacks/
    └── my-pack.zip
```

- 单层顶层目录自动展平
- 不限制目录名，按相对路径存储

## 三、文件清单格式

```json
{
  "version": "v1.2.0",
  "created_at": "2026-04-29T09:20:00+08:00",
  "files": [
    {"path": "mods/sodium.jar", "sha1": "...", "sha256": "...", "size": 5120000}
  ],
  "file_count": 38,
  "total_size": 50200000
}
```

## 四、新旧兼容

- 旧版 CLI 导入（`starter pack import`）仍可用
- 新增 HTTP API 导入（`POST /admin/packs/.../import`）
- 底层都是 `pack.ImportZip`
