# 多包管理手册

> 新增文档 — 2026-04-30

---

## 一、概念

| 术语 | 说明 |
|------|------|
| 主包（Primary Pack） | 服务端配置的唯一默认包。客户端开箱即用，自动同步 |
| 副包（Secondary Pack） | 非主包。客户端不主动管理，需用户手动启用 |
| 启用 | 将副包加入同步列表，开始接受增量更新 |
| 禁用 | 从同步列表移除，文件保留不删 |
| 卸载 | 整个包目录移到回收站，仅「禁用+目录存在」时可操作 |
| 包状态 | `synced` / `updating` / `disabled` / `none`（未安装） |

## 二、服务端侧

管理员 publish 时指定 primary：
```bash
mc-starter-server pack publish --name main-pack --primary
mc-starter-server pack publish --name extra-content
```

一个服务端只能有一个主包（publish --primary 会替换现有的 primary 标记）。

## 三、客户端侧

### 初次启动

```
starter update (首次)
  → 同步主包完成
  → TUI 提示: "服务端有可用副包，请到托盘菜单启用"
  → 不主动下载任何副包
```

### 托盘中

```
托盘图标 → 右键菜单
├── 更新 [主包名]
├──────────
├── 修复
│   ├── [主包名]
│   │   ├── 修复全部
│   │   ├── 仅清理 mods
│   │   ├── 仅清理 config
│   │   ├── 仅重装 Loader
│   │   └── 回滚到上一版本
│   ├── [副包A]  (如已启用)
│   │   └── (修复子菜单)
│   └── [副包B]  (如已启用)
│       └── (修复子菜单)
├──────────
├── 其他包
│   ├── [副包A]  [启用✓]    ← 点击切换
│   ├── [副包B]  [禁用]
│   │   ├── 启用             ← 点击启用
│   │   └── 卸载             ← 移到回收站（仅禁用+目录存在）
│   └── [副包C]  [禁用]
├──────────
└── 退出
```

### 启用/禁用/卸载规则

| 操作 | 条件 | 动作 |
|------|------|------|
| 启用 | 副包禁用状态 | 加入同步列表，下次 update 开始同步 |
| 禁用 | 已启用 | 从同步列表移除，文件保留 |
| 卸载 | 禁用 + 目录存在 | 移到回收站，按钮变灰 |

### CLI 命令

```bash
# 默认更新主包
starter update

# 更新指定副包
starter update --pack extra-content

# 更新所有已启用的包
starter update --all

# 修复指定包
starter repair main-pack --mods-only
```

## 四、包间关系

- 不同包的 mods/config/resourcepacks 在各自 `packs/{name}/` 目录下
- 不互相覆盖，不合并
- 启动器（PCL2/HMCL）的版本隔离已保证不同目录的 mods 各自独立加载
- mc-starter 只负责把文件下载到对应目录，不负责写入 MC 版本目录
