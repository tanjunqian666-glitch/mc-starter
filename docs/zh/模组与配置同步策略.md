# 模组与配置同步策略（更新：包独立目录）

> 原方案中模组同步到 `.minecraft/mods/` → 改为同步到 `.minecraft/packs/{包名}/mods/`
> 详见 `多包管理手册.md`

---

## 一、目录结构

```
.minecraft/
├── packs/
│   ├── main-pack/
│   │   ├── mods/
│   │   └── config/
│   └── extra-content/
│       └── mods/
├── versions/
├── assets/
└── libraries/
```

## 二、同步策略

| 文件类型 | 源 | 目标 |
|---------|-----|------|
| 模组 (.jar/.disabled) | 服务端 import 的 zip 中的 `mods/` | `.minecraft/packs/{包名}/mods/` |
| 配置 (.properties/.json/.toml) | 服务端 `config/` | `.minecraft/packs/{包名}/config/` |
| 资源包 (.zip) | 服务端 `resourcepacks/` | `.minecraft/packs/{包名}/resourcepacks/` |

## 三、增量同步机制

```
客户端已有一个版本的 mods/config → 拉增量 → 只变更：
  新增：下载到目标目录
  更新：覆盖旧文件
  删除：移除文件
  未变：跳过
```

利用 CacheStore 按 SHA256 去重，跨版本复用的文件直接复制不下载。
