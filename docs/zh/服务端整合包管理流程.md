# MC 版本更新器 — 服务端整合包管理流程

> 服务端具备解包、解析、差异分析、版本发布的能力
> 发布者只需上传 zip，服务端自动完成后续所有工作

---

## 一、整体流程

```
┌──────────────────────────────────────────────────────────────┐
│                        发布者                                │
│  PCL2 配好整合包 → 打包 zip → 上传到服务端管理后台           │
└─────────────────────────┬────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────┐
│                        服务端                                │
│                                                              │
│  ① 解包：解析 zip 内容                                       │
│     └→ 提取所有文件清单（路径、hash、大小）                   │
│                                                              │
│  ② 识别：分析包中的模组和配置                                 │
│     ├─ 扫描 mods/*.jar → 计算 Modrinth/CurseForge hash       │
│     │  → 匹配数据库 → 获取模组名称、版本、作者               │
│     ├─ 扫描 config/* → 记录配置文件清单                       │
│     ├─ 提取 MC 版本和 Loader 版本（如果有 manifest 信息）     │
│     └─ 提取 pack.png 图标                                     │
│                                                              │
│  ③ 对比：与上一版本做差异分析                                 │
│     └→ 新增了哪些模组、删除了哪些、更新了哪些                │
│                                                              │
│  ④ 发布：生成待发布版本                                       │
│     ├─ 生成新的 server.json（或发布清单）                     │
│     ├─ 计算增量（客户端只需下载变动的部分）                   │
│     └─ 推送到 CDN / 镜像源                                    │
│                                                              │
└─────────────────────────┬────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────┐
│                        客户端                                │
│  请求更新 → 对比本地版本 → 增量同步 → 完成                    │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 解包分析

```go
// 服务端解包分析工具（也可作为 CLI 工具独立运行）
func analyzePackZip(zipPath string) (*PackAnalysis, error) {
    reader, err := zip.OpenReader(zipPath)
    if err != nil {
        return nil, fmt.Errorf("无法打开 zip: %w", err)
    }
    defer reader.Close()
    
    analysis := &PackAnalysis{
        Mods:    []ModEntry{},
        Configs: []ConfigEntry{},
    }
    
    for _, f := range reader.File {
        if f.FileInfo().IsDir() {
            continue
        }
        
        switch {
        case strings.HasPrefix(f.Name, "mods/") && strings.HasSuffix(f.Name, ".jar"):
            // 分析模组文件
            mod := analyzeModJar(f)
            analysis.Mods = append(analysis.Mods, mod)
            
        case strings.HasPrefix(f.Name, "config/"):
            // 记录配置文件
            hash := computeEntryHash(f)
            analysis.Configs = append(analysis.Configs, ConfigEntry{
                Path: f.Name,
                Hash: hash,
                Size: f.UncompressedSize64,
            })
            
        case f.Name == "pack.png":
            // 提取图标
            analysis.IconHash = computeEntryHash(f)
            
        case strings.HasPrefix(f.Name, "resourcepacks/"):
            analysis.ResourcePacks = append(analysis.ResourcePacks, f.Name)
            
        case strings.HasPrefix(f.Name, "shaderpacks/"):
            analysis.ShaderPacks = append(analysis.ShaderPacks, f.Name)
        }
    }
    
    // 尝试识别 MC 版本和 Loader
    analysis.MCVersion, analysis.Loader = detectVersionFromMods(analysis.Mods)
    
    return analysis, nil
}
```

// 数据库来源：Modrinth API / CurseForge API / 本地 ModData 缓存

func analyzeModJar(f *zip.File) ModEntry {
    content, _ := readZipEntry(f)
    sha1 := sha1Hash(content)
    
    entry := ModEntry{
        FileName: filepath.Base(f.Name),
        SHA1:     sha1,
        Size:     f.UncompressedSize64,
    }
    
    // 尝试匹配 Modrinth API
    modrinthInfo := matchModrinthByHash(sha1)
    if modrinthInfo != nil {
        entry.Name = modrinthInfo.Name
        entry.Version = modrinthInfo.Version
        entry.Source = "modrinth"
        entry.Known = true
        return entry
    }
    
    // 尝试匹配本地数据库（PCL 的 ModData.txt 格式）
    localInfo := matchLocalDatabase(sha1)
    if localInfo != nil {
        entry.Name = localInfo.Name
        entry.Source = "local_db"
        entry.Known = true
        return entry
    }
    
    // 未知模组
    entry.Name = filepath.Base(f.Name)
    entry.Known = false
    return entry
}

### 3.2 差异分析

```go
func diffVersions(previous, current *PackManifest) *VersionDiff {
    diff := &VersionDiff{}
    
    prevFiles := make(map[string]string) // path → hash
    for _, f := range previous.Files {
        prevFiles[f.Path] = f.Hash
    }
    
    currFiles := make(map[string]string)
    for _, f := range current.Files {
        currFiles[f.Path] = f.Hash
    }
    
    // 新增：当前有，上一版没有
    for path, hash := range currFiles {
        if _, ok := prevFiles[path]; !ok {
            diff.Added = append(diff.Added, FileChange{Path: path, Hash: hash})
        }
    }
    
    // 删除：上一版有，当前没有
    for path, hash := range prevFiles {
        if _, ok := currFiles[path]; !ok {
            diff.Removed = append(diff.Removed, FileChange{Path: path, Hash: hash})
        }
    }
    
    // 更新：都有但 hash 不同
    for path, currHash := range currFiles {
        if prevHash, ok := prevFiles[path]; ok && currHash != prevHash {
            diff.Updated = append(diff.Updated, FileChange{
                Path:     path,
                OldHash:  prevHash,
                NewHash:  currHash,
            })
        }
    }
    
    // 统计分析
    diff.Stats = DiffStats{
        AddedSize:   calcTotalSize(diff.Added, current),
        RemovedSize: calcTotalSize(diff.Removed, previous),
        UpdatedSize: calcTotalSize(diff.Updated, current),
        NetChange:   len(diff.Added) + len(diff.Updated) - len(diff.Removed),
    }
    
    return diff
}
```

### 4.2 客户端升级策略

```go
func checkUpdate(cfg *ClientConfig, localVersion string) error {
    // 请求服务端
    updateInfo, err := fetchUpdateInfo(cfg.ServerURL, localVersion)
    if err != nil {
        return err
    }
    
    switch updateInfo.UpdateMode {
    case "none":
        fmt.Println("已是最新版本")
        return nil
        
    case "incremental":
        // 增量更新：只下载变动部分
        // 适合大部分场景
        applyIncrementalUpdate(updateInfo.Incremental)
        
    case "full":
        // 全量更新：下载完整 zip
        // 触发条件：
        //   - MC 版本或 Loader 变了
        //   - 客户端本地没有旧版本快照
        //   - 增量差异过大（超过 50% 全量大小）
        downloadFullPack(updateInfo.FullPack)
    }
    
    return nil
}

// 服务端判断使用哪种更新模式
func decideUpdateMode(prevVersion, currVersion string, diff *VersionDiff) string {
    // MC 版本或 Loader 变更 → 全量
    if prevVersion.MCVersion != currVersion.MCVersion {
        return "full"
    }
    if prevVersion.Loader != currVersion.Loader {
        return "full"
    }
    
    // 增量差异超过全量 50% → 全量
    if diff.Stats.NetDownload > diff.Stats.FullPackSize * 0.5 {
        return "full"  
    }
    
    // 没有差异 → 无需更新
    if len(diff.Added) == 0 && len(diff.Removed) == 0 && len(diff.Updated) == 0 {
        return "none"
    }
    
    return "incremental"
}
```

### 4.3 旧版本传统发布模式兼容

**传统模式**（无增量、无差异分析）也同时保留：

```json
{
  "version": "1.2.0",
  "pack_url": "https://cdn.你的服务器/packs/cjc-pack-v1.2.0.zip",
  "pack_hash": "sha256:...",
  "pack_size": 52428800,
  "mc_version": "1.20.1",
  "loader": {
    "type": "fabric",
    "version": "0.15.11"
  }
}
```

两种模式互不冲突：
- **第一次部署**：用传统模式（全量 zip），客户端啥都没有只能全量下
- **后续更新**：客户端有快照了，自动走增量
- **服务端不想折腾**：只用传统模式也行，增量是可选的增强

## 七、简化方案

如果不需要完整的 Web 后台，最简的工作流可以是：

```
1. 压缩包丢到服务器的一个目录
2. 运行 ./pack-analyzer publish 整合包.zip
3. 程序自动完成：
   ├─ 解包分析
   ├─ 与上一版本 diff
   ├─ 生成增量清单
   ├─ 更新 server.json
   └─ 发布到 CDN

4. 客户端下次请求时自动拿到增量更新
```

这个 `pack-analyzer` 就是一个 Go 写的 CLI 工具，跑在服务端，和客户端的 `starter` 独立。两者共享同样的 hash 算法和 Manifest 数据结构。
