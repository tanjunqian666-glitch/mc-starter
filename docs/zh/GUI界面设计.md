# GUI 界面设计

> 2026-04-30 · 替换原 bubbletea TUI

---

## 设计哲学

mc-starter 的 GUI 定位是 **小工具**，不是大应用：

- **固定小窗口** ~380×230，不可缩放
- **双击即用**：自动检测配置，无配置则弹出向导
- **隐藏复杂度**：用户不需要知道 Fabric/Forge/内存/备份/缓存
- **Windows 原生**：基于 lxn/walk，单 exe 发布

## 主窗口

```
┌─ MC Starter ────────────────[⚙]─┐
│                                    │
│  版本: [主整合包 v1.2.0       ▼]   │
│                                    │
│  [📂 打开启动器]  [🔄 更新]       │
│                                    │
│  主整合包  本地: v1.2.0  最新: v1.3.0
│  有可用更新                        │
└────────────────────────────────────┘
```

### 控件说明

| 控件 | 作用 |
|------|------|
| ⚙ | 打开设置弹窗 |
| 版本下拉 | 显示服务端返回的所有版本（主版本排第一，副版本在后） |
| 打开启动器 | 同步完成才能点击；未安装时提示"请先更新" |
| 更新 | 有可用更新时才亮；点击触发同步+自动装加载器 |
| 版本状态栏 | 格式：`版本名  本地: x  最新: y` |
| 状态文字 | 就绪/有可用更新/已是最新/正在同步... |

### 状态流转

```
无版本信息 → "就绪"
版本未安装 → "未安装，请点击更新"（橙色）
已安装有更新 → "有可用更新"（橙色）
已安装无更新 → "已是最新"（绿色）
同步中 → "正在同步..."（橙色）
同步取消 → "已取消"（橙色）
```

## 首次配置向导

首次双击弹出 3 步向导：

| 步骤 | 内容 | 自动行为 |
|------|------|---------|
| 1 | 填写服务器 API 地址 | 验证非空 |
| 2 | 启动器路径 | 自动搜索 PCL2/HMCL，🔍 重新检测，📁 手动选文件 |
| 3 | Minecraft 根目录 | 自动搜索常见位置，🔍 重新检测，📁 手动选目录 |
| 完成 | 保存配置 + 拉取服务端版本列表 | 回到主窗口 |

## 同步过程

- 用户点击更新 → 按钮灰 → 进度条显示
- 进度条仅显示百分比（0-100%）
- 取消按钮 → 中断同步 → 自动回滚（复用 repair 模块的 Rollback）
- 同步完成 → 更新本地版本号 → 自动刷新 UI

## 技术实现

**框架**：`github.com/lxn/walk` — Go 原生 Windows GUI 库

**文件结构**：
```
internal/gui/
├── app.go       ← 主窗口 + 布局 + 操作逻辑
├── settings.go  ← 设置弹窗
└── setup.go     ← 首次配置向导 + 自动检测工具函数
```

**依赖变化**：从 bubbletea（~15 依赖）→ walk（~5 依赖），exe 体积缩小约 200KB
