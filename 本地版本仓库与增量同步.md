# MC 版本更新器 — 本地版本仓库与增量同步

> 在客户端维护一个完整副本仓库，后续更新只传增量，节省服务器带宽

---

## 一、设计思路

### 核心概念

```
每次全量下载后，本地保留一份"快照副本"
下次更新时对比快照 → 找出增量 → 只下载变动的部分
```

```
┌──────────────────────────────────────────────┐
│                服务端                          │
│  server.json（模组清单 + hash）                │
│  mod-1.0.jar  ← 当前版本                       │
│  mod-1.1.jar  ← 更新后版本                     │
└──────────────┬───────────────────────────────┘
               │
               ▼ 用户请求更新
┌──────────────────────────────────────────────┐
│                客户端                          │
│                                                │
│  .minecraft/mods/  ← 当前使用中的模组          │
│                                                │
│  .minecraft/starter_repo/  ← 本地仓库          │
│   ├── v1.0/              ← 首次全量后的快照    │
│   │   ├── mod-1.0.jar                          │
│   │   ├── mod-1.1.jar                          │
│   │   └── manifest.json                        │
│   ├── v1.1/              ← 增量更新后的快照    │
│   │   └── manifest.json   （只变了的文件）     │
│   └── current/ ← symlink / 指向当前版本        │
│                                                │
└──────────────────────────────────────────────┘

更新时：
  对比 server.json 清单 vs v1.0 manifest
  → 新增的文件：下载
  → 变动的文件：下载
  → 删除的文件：本地删除
  下载后生成 v1.1 snapshot（可以被下次增量复用）
```

## 三、增量同步流程

### 3.1 几种同步模式对比

| 模式 | 行为 | 带宽消耗 |
|---|---|---|
| 全量下载 | 不管本地有没有，全下一遍 | 高 |
| 差异化同步（已有） | 只下本地没有的/更新的 | 中 |
| **版本仓库增量** | 仓库有缓存就直接用，只下新增的 | **低** |

### 3.2 完整增量同步算法

```go
func incrementalSync(cfg *ServerConfig, mcDir string, repo *LocalRepo) error {
    serverManifest := fetchServerManifest(cfg)
    latestSnapshot := repo.GetLatestSnapshot()
    
    if latestSnapshot == nil {
        // 首次同步：全量下载 + 建立仓库
        return fullDownloadAndCreateRepo(cfg, mcDir, repo)
    }
    
    // 检测版本是否变了
    if latestSnapshot.MCVersion != cfg.MCVersion {
        // MC 大版本变更 → 无法增量，需要全量
        fmt.Println("MC 版本从", latestSnapshot.MCVersion, "变更为", cfg.MCVersion)
        fmt.Println("需要全量下载")
        return fullDownloadAndCreateRepo(cfg, mcDir, repo)
    }
    
    // 差异计算
    changes := computeChanges(serverManifest, latestSnapshot)
    
    if len(changes.Add) == 0 && len(changes.Update) == 0 && len(changes.Delete) == 0 {
        fmt.Println("你已经是最新版本了 ✓")
        return nil
    }
    
    // 显示变更
    printChanges(changes)
    
    // 创建新快照（从旧快照复制 + 应用变更）
    newSnapshot := repo.CreateSnapshot(cfg.Version)
    newSnapshot.CopyFrom(latestSnapshot)
    
    // 处理变更
    for _, f := range changes.Add {
        // 检查缓存
        cachedPath := repo.GetCachedFile(f.Hash)
        if cachedPath != "" {
            // 缓存命中，直接复制
            copyFile(cachedPath, filepath.Join(mcDir, "mods", f.Name))
            fmt.Printf("  📦 [缓存] %s\n", f.Name)
        } else {
            // 下载
            download(f.URL, filepath.Join(mcDir, "mods", f.Name), f.Hash)
            // 加入缓存
            repo.CacheFile(f.Hash, filepath.Join(mcDir, "mods", f.Name))
            fmt.Printf("  📥 [下载] %s\n", f.Name)
        }
        newSnapshot.AddFile(f)
    }
    
    for _, f := range changes.Update {
        oldHash := latestSnapshot.GetFile(f.Name).Hash
        cacheKey := oldHash + "_to_" + f.Hash
        
        // 检查是否有预计算的 diff
        diffPath := repo.GetDiff(cacheKey)
        if diffPath != "" && false { // diff 方案暂不实现
            // apply diff
        } else {
            // 全量更新（但源文件可能在缓存中）
            if repo.HasCachedFile(f.Hash) {
                copyFile(repo.GetCachedFile(f.Hash), filepath.Join(mcDir, "mods", f.Name))
                fmt.Printf("  🔄 [缓存更新] %s\n", f.Name)
            } else {
                download(f.URL, filepath.Join(mcDir, "mods", f.Name), f.Hash)
                repo.CacheFile(f.Hash, filepath.Join(mcDir, "mods", f.Name))
                fmt.Printf("  🔄 [下载更新] %s\n", f.Name)
            }
        }
        newSnapshot.UpdateFile(f)
    }
    
    for _, f := range changes.Delete {
        os.Remove(filepath.Join(mcDir, "mods", f.Name))
        fmt.Printf("  🗑  [删除] %s\n", f.Name)
        newSnapshot.RemoveFile(f.Name)
    }
    
    // 保存新快照
    repo.SaveSnapshot(newSnapshot)
    repo.SetCurrent(newSnapshot.Version)
    
    // 显示统计
    fmt.Printf("\n✅ 增量同步完成\n")
    fmt.Printf("  新增: %d, 更新: %d, 删除: %d\n", 
        len(changes.Add), len(changes.Update), len(changes.Delete))
    fmt.Printf("  实际下载: %d 个文件（缓存命中: %d 个）\n",
        len(changes.Add)+len(changes.Update)-cacheHits, cacheHits)
    fmt.Printf("  仓库大小: %s\n", formatBytes(repo.GetSize()))
    
    return nil
}
```

### 3.3 变更计算

```go
type Changes struct {
    Add    []FileEntry
    Update []FileEntry
    Delete []FileEntry
}

func computeChanges(server, local *Snapshot) Changes {
    var c Changes
    
    serverFiles := make(map[string]FileEntry)
    for _, f := range server.Files {
        serverFiles[f.Name] = f
    }
    
    localFiles := make(map[string]FileEntry)
    for _, f := range local.Files {
        localFiles[f.Name] = f
    }
    
    // 新增：服务端有，本地没有
    for name, f := range serverFiles {
        if _, ok := localFiles[name]; !ok {
            c.Add = append(c.Add, f)
        }
    }
    
    // 更新：都有的文件 hash 不同
    for name, sf := range serverFiles {
        if lf, ok := localFiles[name]; ok {
            if sf.Hash != lf.Hash {
                c.Update = append(c.Update, sf)
            }
        }
    }
    
    // 删除：本地有，服务端没有
    for name, f := range localFiles {
        if _, ok := serverFiles[name]; !ok {
            c.Delete = append(c.Delete, f)
        }
    }
    
    return c
}
```

## 四、文件缓存

### 4.1 缓存策略

```
所有下载过的文件，按 hash 名存到 starter_repo/files/
相同 hash 的文件不重复下载
```

```go
type FileCache struct {
    repoDir string
}

// 文件缓存路径 = hash 的前 2 位做子目录 + 完整 hash 做文件名
// 避免一个目录下文件太多
func (c *FileCache) cachePath(hash string) string {
    hashOnly := strings.TrimPrefix(hash, "sha256:")
    subdir := hashOnly[:2]
    return filepath.Join(c.repoDir, "files", subdir, hashOnly)
}

// 尝试从缓存获取文件
func (c *FileCache) Get(hash string) (string, bool) {
    path := c.cachePath(hash)
    if _, err := os.Stat(path); err == nil {
        // 校验缓存完整性
        if verifyHash(path, hash) {
            return path, true
        }
        // 缓存损坏，删了重新下
        os.Remove(path)
    }
    return "", false
}

// 加入缓存
func (c *FileCache) Put(hash string, srcPath string) error {
    dstPath := c.cachePath(hash)
    os.MkdirAll(filepath.Dir(dstPath), 0755)
    return copyFile(srcPath, dstPath)
}
```

### 4.2 缓存收益

```
场景：同一个模组在多版本间未变
  v1.0: sodium-0.5.3.jar (hash: AAAA)
  v1.1: sodium-0.5.3.jar (hash: AAAA) ← 没变，直接复用

场景：同一台机器装多个整合包
  整合包 A: lithium-0.11.2.jar (hash: BBBB) → 已缓存
  整合包 B: lithium-0.11.2.jar (hash: BBBB) → 已缓存 ✓
```

### 4.3 缓存清理

```go
// 每 30 天检查一次缓存
// 删除那些没有任何快照引用的孤立文件
func (c *FileCache) Cleanup() {
    // 收集所有快照引用的 hash
    referencedHashes := c.collectReferencedHashes()
    
    // 扫描文件缓存目录
    for _, f := range c.listCachedFiles() {
        if !referencedHashes[f.Hash] {
            // 没有被任何快照引用 → 删除
            os.Remove(f.Path)
            log("清理孤立缓存: %s", f.Hash[:16])
        }
    }
}
```

## 五、快照管理

### 5.1 快照链

```
v1.0 ──→ v1.1 ──→ v1.2
  │         │         │
  ├ mods    ├ mods    ├ mods ← 每个快照独立记录文件清单
  ├ configs ├ configs ├ configs
  └ 完整     └ 增量    └ 增量
```

v1.0 是全量快照（记录所有文件）
v1.1, v1.2 是增量快照（只记录变动的文件）
**要还原 v1.1 的完整状态** = v1.0 所有文件 + v1.1 的增量

### 5.2 快照恢复

```go
// 根据快照版本还原 .minecraft/mods/ 等内容
func restoreFromSnapshot(mcDir string, snapshotVersion string, repo *LocalRepo) error {
    // 找到从 v1.0 到此版本的全部增量
    snapshots := repo.GetSnapshotsUpTo(snapshotVersion)
    
    // 清空当前 mods/
    os.RemoveAll(filepath.Join(mcDir, "mods"))
    os.MkdirAll(filepath.Join(mcDir, "mods"), 0755)
    
    // 从快照链重建
    currentState := make(map[string]FileEntry)
    
    for _, snap := range snapshots {
        for name, file := range snap.Files {
            switch file.Action {
            case "add", "update":
                // 从缓存复制
                if cachedPath, ok := repo.Cache.Get(file.Hash); ok {
                    copyFile(cachedPath, filepath.Join(mcDir, "mods", name))
                    currentState[name] = file
                }
            case "delete":
                delete(currentState, name)
            }
        }
    }
    
    // 写回 snapshot.current 指向
    repo.SetCurrent(snapshotVersion)
    
    return nil
}
```

### 6.2 客户端升级路径计算

```go
// 从当前版本升级到目标版本的路径
func calcUpgradePath(localVersion string, serverCfg *ServerConfig) UpgradePath {
    if localVersion == "" {
        // 没有本地版本 → 全量
        return UpgradePath{Type: "full"}
    }
    
    if serverCfg.Repo.Version == localVersion {
        // 已是最新 → 无变化
        return UpgradePath{Type: "none"}
    }
    
    // 收集需要应用的增量
    var increments []string
    found := false
    for _, v := range serverCfg.Repo.Snapshots {
        if v.IncrementalFrom == localVersion {
            found = true
        }
        if found {
            increments = append(increments, v.Version)
        }
    }
    
    return UpgradePath{
        Type:       "incremental",
        From:       localVersion,
        To:         serverCfg.Repo.Version,
        Snapshots:  increments,
    }
}
```

### 6.3 增量清单示例

```
服务端提供 v1.2 增量清单（相当于只发改变的东西）：
  ［新增］mod-optimization-1.1.jar
  ［更新］config/sodium-mixins.properties（从 hash: old → hash: new）
  ［删除］mod-obsolete-1.0.jar

客户端拿到增量清单后：
  - 新增 → 下载
  - 更新 → hash 不同才下
  - 删除 → 直接删本地

所有已下过的文件触发缓存命中 → 网络走了 0 KB
```

## 七、带宽节省估算

### 7.1 典型场景

```
整合包总大小：~50 MB（15 个模组 + 配置）
首次全量下载：50 MB

后续每次变动：1~3 个模组
  无缓存情况下：~5 MB（只下载变动的）
  有缓存情况下：~0 KB（文件没变，只同步 metadata）

一个月更新 4 次：
  无仓库：50 MB × 4 = 200 MB/月
  有仓库：50 MB + 5 MB × 3 = 65 MB/月
  有仓库+缓存：50 MB + 0 MB × 3 = 50 MB/月（只有首次）

节省：67% ~ 75%
```

### 7.2 多整合包场景

```
用户有 3 个整合包：
  整合包 A（模组 15 个，部分与 B 重合）
  整合包 B（模组 12 个）
  整合包 C（模组 20 个）

总面积估算：~300 MB（重复下载）
  无缓存：300 MB
  有缓存（共享相同文件）：~180 MB（节省 40%）
```

## 八、文件去重（跨整合包复用）

```go
// 全局缓存（独立于 .minecraft 之外）
// 放在 %APPDATA%/mc-starter-cache/ 或 ~/.mc-starter-cache/
//
// 无论哪个整合包下载的文件，只要 hash 相同就复用
// sodium-0.5.3.jar 在整合包 A 下了一次，整合包 B 直接复制

type GlobalCache struct {
    cacheDir string
}

func NewGlobalCache() *GlobalCache {
    cacheDir := filepath.Join(getAppDataDir(), "mc-starter-cache")
    os.MkdirAll(cacheDir, 0755)
    return &GlobalCache{cacheDir: cacheDir}
}

func (c *GlobalCache) Get(hash string) (string, bool) {
    // hash 做文件名，全局唯一
    path := filepath.Join(c.cacheDir, hashToFilename(hash))
    if _, err := os.Stat(path); err == nil {
        return path, true
    }
    return "", false
}
```

## 九、与修复工具的关系

| 场景 | 修复模式 | 仓库行为 | 带宽 |
|---|---|---|---|
| 日常更新 | `sync`（差异同步） | 创建增量快照 | 极低 |
| 模组崩了 | `repair --mods-only` | 清空 mods 然后从快照链重建 | 0（本地缓存） |
| 完全修复 | `repair --clean` | 从最近快照重建 | 0（本地缓存） |
| 配置坏了 | `repair --config-only` | 从最近快照复制配置 | 0（本地缓存） |

```
有仓库后，"修复" = 从本地快照还原，0 带宽消耗
只有"下载新模组"才消耗带宽
```
